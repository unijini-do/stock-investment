# 공통 출력 계약

## Markdown

분석 원문과 판단 근거를 완전하게 보존하는 기준 문서다.

필수 섹션:

1. 요약
2. 핵심 데이터
3. 분석과 해석
4. 강세 시나리오
5. 반대 시나리오와 무효화 조건
6. 리스크와 사용자 결정 게이트
7. 다음 단계
8. 출처

## HTML

- `templates/dashboard.html`의 CSS 변수와 카드 구조를 사용한다.
- 단일 파일로 열 수 있어야 한다.
- Markdown과 결론, 숫자, 출처가 일치해야 한다.
- 진입 및 추가매수는 파랑, 목표 및 익절은 초록, 손절 및 위험은 빨강으로 표시한다.
- 모바일 우선이며 최대 폭은 1080px, 기본 분석은 680px 안팎의 세로형 구성을 권장한다.
- JavaScript 없이도 핵심 내용을 읽을 수 있어야 한다.

## JSON

- 각 에이전트 지침의 입출력 계약을 유지한다.
- 불명확하거나 미확인인 필드는 빈 값으로 꾸미지 말고 `null` 또는 `"데이터 미확인"`으로 기록한다.
- 날짜는 `YYYY-MM-DD`, 시각은 ISO 8601을 사용한다.
- 가격과 비율은 가능하면 문자열이 아니라 숫자로 기록하고 통화 및 단위를 별도 필드로 둔다.

## 동기화

세 형식의 핵심 결론과 숫자는 반드시 동일해야 한다. 수정할 때는 Markdown, HTML, JSON을 함께 갱신한다.
