# 대시보드 디자인 시스템 토큰

루트의 `templates/dashboard.html`을 복제·치환하는 것이 원칙. 아래는 토큰 빠른 참조.

## 색상 (CSS 변수)
| 의미 | 배경 | 텍스트 | 용도 |
|---|---|---|---|
| 🟢 강세·매수·핵심 | #DCEFE0 | #1C7A3E | 매수 신호, 익절가, 강세 목표 |
| 🔵 기본·중립우호·수혜 | #DEE8FB | #2B5BA8 | 수혜 기업, 진입가, 기본 시나리오 |
| 🔴 약세·리스크·손절 | #FBE1E1 | #C53A2F | 리스크, 손절가, 약세 시나리오 |
| 🟡 경계·약한 하락 | #FBF0D6 | #9A6B12 | 경계 신호 |
| ⚪ 중립 | #EDEEF0 | #5A5A5A | 타임프레임 등 중립 정보 |

## 레이아웃
- 폰트: Pretendard (CDN 링크 포함)
- 배경 #FBFBFA / 카드 #FFF + 보더 #EAE9E4 / 헤더카드 크림 #F4F2EA
- 둥근 모서리 14~18px, 카드 간격 18px, 최대폭 1080px

## 방향/강도 기호
▲ 강세강 · △ 강세약 · ▬ 보합 · ▽ 약세약 · ▼ 약세강

## 가격 레벨 박스 (필수 색상 구분)
- 진입가 = 파랑 박스 (.lvl-entry)
- 익절가 = 초록 박스 (.lvl-tp)
- 손절가 = 빨강 박스 (.lvl-sl)
- 모든 숫자는 tabular-nums (.num 클래스)

## 6개 보드 순서 (고정)
헤더 → 거시리스크 → 주도산업·핵심기술 → 이벤트시나리오 → 종목분석카드 → 리스크관리패널

## 치환 규칙
- 템플릿의 `[대괄호]` 자리표시자를 실제 데이터로 교체.
- 종목이 여러 개면 `.stock` 블록을 복제해 자산 수만큼 반복.
- 역발상 3선·수혜기업 3사·리스크 3순위는 행/카드 수를 고정(3개).
- 데이터 미확인 항목은 "데이터 미확인"으로 표기하고 빈칸으로 두지 않는다.
